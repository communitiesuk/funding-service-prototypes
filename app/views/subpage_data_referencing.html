Next steps

To monitor after release:

  • How does the 'Format data' page perform with real data sets? Would radios be more user-friendly and more accessible than select components?
  • How many columns and how many rows do real data sets have? What is the impact on the 'View data set' page?
  • How are data types and data formatting (prefixes, suffixes and decimals) used with real data?
  • On the 'Format data' page, how do form designers use the 'Example data' column?
  • On the 'View data set' page, consider adding the report status.
  • What is the overall experience like for form designers uploading CSVs like with real data sets? Would it help to be able to set up the schema in the service before uploading CSV data?
  • What is the overall experience like for correcting data? Would it help to be able to correct individual values in the service without having to replace CSV data?
  • Add functionality for the user to be able to change a data set name without having to delete or replace a data set.
